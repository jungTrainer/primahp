# Firebase 비용 안전 운영 가이드

## 기본 운영 원칙
- 초기 운영은 결제수단이 필요 없는 Firebase Spark 플랜으로 시작합니다.
- 방문자용 공개 콘텐츠는 `data/*.json` 정적 파일에서 제공합니다.
- Firestore는 입원문의와 보호자 댓글처럼 서버 저장이 필요한 쓰기 기능에만 사용합니다.
- 관리자 CRM만 Firestore 읽기를 사용하며, 공개 페이지에서는 Firestore 실시간 구독을 사용하지 않습니다.

## 공개 데이터 구조
- `data/notices.json`: 공지사항
- `data/staff.json`: 의료진 및 조직 정보
- `data/gallery.json`: 갤러리
- `data/site-content.json`: 홈페이지 문구

정적 파일은 웹 호스팅/CDN 캐시를 활용하므로 방문자가 증가해도 Firestore 읽기 사용량이 증가하지 않습니다.

## 관리자 CRM 조회 원칙
- `onSnapshot()`은 꼭 필요한 관리 화면에서만 사용합니다.
- 문의·댓글 목록은 최신 항목만 조회하도록 `orderBy()`와 `limit()`을 적용합니다.
- 대시보드 기본 조회 범위는 최근 30일 또는 최신 50건으로 제한합니다.
- 과거 데이터는 페이지네이션 또는 기간 검색을 통해 추가 조회합니다.
- 관리 화면을 닫거나 다른 화면으로 이동할 때 실시간 리스너의 unsubscribe 함수를 반드시 실행합니다.
- 전체 컬렉션을 제한 없이 읽는 쿼리는 사용하지 않습니다.

## Blaze 전환 시 안전장치
- Google Cloud Billing 예산과 이메일 알림을 설정합니다.
- 예산 알림은 비용을 자동 차단하지 않으므로 사용량 모니터링과 쿼리 제한을 함께 적용합니다.
- Firestore 사용량 대시보드에서 읽기·쓰기·삭제·저장량을 정기적으로 확인합니다.
- 예상 사용량을 넘으면 공개 읽기 경로와 관리자 쿼리를 우선 점검합니다.

## 배포 전 점검
- 공개 페이지에서 Firestore `onSnapshot()`이 실행되지 않는지 확인
- 관리자 목록 쿼리에 `limit()`이 적용됐는지 확인
- 실시간 리스너 해제 처리가 있는지 확인
- 문의·댓글 쓰기 실패 시 성공 메시지가 표시되지 않는지 확인
- Firestore Security Rules가 배포됐는지 확인
- 테스트용 개인정보와 예시 전화번호가 제거됐는지 확인
